# 콘텐츠 건강성 가이드 (Content Safety Guide)

> 외부 플랫폼(유튜브·인스타·틱톡·스레드·블로그)에서 계정 제한 없이
> 안전하게 노출할 수 있는 콘텐츠 기준을 정리합니다.
> 이 가이드는 PG 심사·광고 심사·플랫폼 정책 모두를 고려합니다.

---

## 1. 외부 플랫폼에서 피해야 할 표현

### ❌ 절대 금지 (계정 차단·광고 거절 위험)

| 피해야 할 표현 | 이유 |
|---|---|
| "사주 봐드립니다" | 미신·점술 분류 위험 |
| "점 봐드립니다" | 미신·점술 분류 위험 |
| "운명 확인" | 운명론적 단정 |
| "재회운 99%" | 수치 단정, 광고법 위반 가능 |
| "궁합운 대박" | 과장 표현, 광고법 위반 |
| "반드시 일어납니다" | 단정적 예언 |
| "이 날짜에 큰일 납니다" | 공포 조장 |
| "전남친 속마음 맞춰드립니다" | 제3자 정보 단정 |
| "외도/잠자리/임신/질병/죽음 단정" | 민감 정보 단정, 법적 리스크 |
| "올해 대박납니다" | 과장 성과 약속 |
| "이 사람 마음이 아직 있습니다" | 제3자 심리 단정 |

### ⚠️ 주의 (맥락에 따라 위험)

- "재회", "복합", "이별 후 연락" — 자사몰 내부에서는 OK, 외부 광고에는 주의
- "사주", "명식", "대운", "세운" — 상품 상세 페이지 안에서는 OK, 광고 제목/썸네일에는 주의
- "궁합" — 자사몰 내부 OK, 외부 노출 시 "관계 패턴"으로 교체

## 2. 외부 플랫폼에서 권장하는 표현

### ✅ 안전하고 효과적인 표현

| 권장 표현 | 적용 상황 |
|---|---|
| 반복 패턴 | 연애/돈/선택 모든 영역 |
| 관계 흐름 | "궁합" 대신 |
| 선택 기준 | "운명" 대신 |
| 마음 정리 | "재회" 주제 접근 시 |
| 돈이 새는 패턴 | 재물운 대신 |
| 이직 전 점검 | 직업운 대신 |
| 오늘의 선택 | 오늘의 운세 대신 |
| 자기이해 리포트 | "사주 풀이" 대신 |
| 관계 체크리스트 | "궁합 확인" 대신 |
| 감정 습관 | 연애 패턴 설명 시 |
| 흐름 분석 | "운세" 대신 |

## 3. 자사몰 내부에서만 사용 가능한 표현

결제 후 진입한 상품 페이지·결과지에서는 아래 표현을 사용할 수 있습니다.
단, 광고 소재·외부 SNS 게시물·검색 메타 설명에는 사용하지 않습니다.

- 사주 명식 / 4기둥 / 년주·월주·일주·시주
- 천간·지지 / 십성 / 대운·세운
- 오늘의 운세 (결과지 섹션 헤딩으로는 OK)
- 재회 흐름 (결과지 본문에서만)
- 궁합 (결과지 내부 분석에서만)

## 8. 검토 체크리스트 (콘텐츠 발행 전)

- [ ] 제목에 "사주/점/운명/운세" 없음
- [ ] 수치·확률 단정 없음 (99%, 반드시, 확실히)
- [ ] 제3자(전남친, 상대방 등)의 마음·행동 단정 없음
- [ ] 건강·임신·금전 성과 확정 표현 없음
- [ ] 공포 조장 표현 없음 (큰일납니다, 조심해야 합니다 → OK, "이 날짜에 큰일 납니다" → NG)
- [ ] 외부 노출 텍스트에서 CTA가 상담형·자기이해형 언어를 사용함

## 9. 자사몰 진입 후 톤 전환 원칙

외부 콘텐츠 → 상품 페이지 진입 시, 갑자기 톤이 바뀌면 신뢰감이 깨집니다.
"외부 라벨 → 상품 페이지 헤드라인 → 결과지 헤딩" 의 톤은 자연스럽게 연결되어야 합니다.

### 9-1. 톤 전환 허용 범위

| 영역 | 사용 가능 단어 | 사용 금지 단어 |
|---|---|---|
| 외부 콘텐츠 (랜딩/contents/광고) | 선택, 흐름, 패턴, 자기이해, 리포트, 체크리스트 | 사주, 명식, 점, 운명, 운세, 점괘, 궁합 |
| 상품 페이지 (외부 → 내부 진입 직후) | 선택, 흐름, 리포트, 진단, 분석, 자기이해 | 점괘, 운명 단정, 수치 단정 |
| 결과지 헤딩 (결제 완료 후) | 사주, 명식, 4기둥, 천간·지지, 십성, 대운, 세운, 오늘의 운세(섹션 헤딩) | 단정적 예언, "반드시", "확실히" |
| 페이월·Sticky CTA | 해금, 열기, 이어보기 | 결제하기, 구매하기, 상품 구매 |

### 9-2. 톤 전환 검수 흐름

```
외부 콘텐츠
  ↓ "오늘의 선택 리포트" 라벨로 클릭
상품 페이지
  ↓ 헤드라인 "오늘 하루, 어떤 선택 흐름을 의식하면 좋을까요?"
    → 외부 톤과 자연스럽게 연결됨
입력 폼 + 결제
  ↓
결과지
  ↓ 결과지 헤딩에서 비로소 "오늘의 운세 한 줄" 같은 내부 표현 허용
  ↓ 단, 본문은 단정적 예언이 아닌 "오늘은 ~한 흐름이 보여요" 톤 유지
```

→ **상품 페이지가 외부와 결과지 사이의 톤 다리** 역할. 갑자기 "사주/명식" 단어를 전면에 세우지 않음.

### 10-1. 매핑 표 검수 원칙

- 외부 라벨과 DB name 사이에 단어 차이가 있어도, 상품 페이지 헤드라인이 두 톤을 자연스럽게 잇는 다리 역할을 해야 함
- `today-fortune` 의 경우 DB name 이 "오늘의 운세 한 줄" 이지만 외부 라벨은 "오늘의 선택 리포트" — 상품 페이지/결과지에서는 "오늘의 선택" 톤 유지하고 "운세" 는 결과지 섹션 헤딩에서만 보조적으로 허용

## 11. 자사몰 내부 결과지/페이월 톤 가이드

### 11-1. 결과지 본문 톤

| 권장 | 금지 |
|---|---|
| "~한 흐름이 보여요" | "반드시 ~합니다" |
| "오늘은 ~을 의식하면 좋아요" | "오늘 ~이 일어날 것입니다" |
| "~한 패턴이 두드러져요" | "~한 일이 99% 확실합니다" |
| "한 박자 기다리는 게 도움이 돼요" | "절대 ~하지 마세요" |

### 11-2. 페이월/CTA 톤 (CLAUDE.md 5 와 동기화)

| 권장 | 금지 |
|---|---|
| 전체 리포트 열기 | 결제하기 |
| 잠긴 N개 항목 해금하기 | 구매하기 |
| 심화 해석 보기 | 상품 구매 |
| 내 결과 이어보기 | 지금 구매 |
| 오늘의 흐름 확인하기 | 정통 사주 구매하기 |
| 오늘의 선택 리포트 열기 | 상품 결제 |

### 11-3. 사용자가 "돈을 낸다" 가 아니라 "내 결과의 다음 장을 연다" 고 느끼게 한다.

## 12. PG 심사 대비 단정·예언·과장 표현 점검

PG 심사·광고법·플랫폼 정책 모두를 통과하기 위해 다음 표현은 어디에서도 사용하지 않습니다.

### 12-1. 단정적 예언
- ❌ "오늘 반드시 좋은 일이 있습니다"
- ❌ "이 사람과 결혼합니다"
- ❌ "올해 안에 이직합니다"
- ✅ "오늘 ~한 흐름이 보여요"
- ✅ "~한 가능성이 있어요"

### 12-2. 수치·확률 단정
- ❌ "재회 가능성 99%"
- ❌ "이 궁합 100% 맞음"
- ❌ "올해 재물운 8점/10점"
- ✅ "재회 흐름의 강도가 보여요"
- ✅ "관계 흐름이 자연스러운 결을 가졌어요"

### 12-3. 민감 영역 단정 (법적 리스크)
- ❌ 질병/죽음/임신/사고 단정 — 어느 영역에서도 금지
- ❌ "이 사람 외도하고 있어요"
- ❌ "이 투자로 N억 법니다"
- ✅ 건강 관련은 "컨디션 관리에 신경 쓰면 좋아요" 정도까지만
- ✅ 재물 관련은 "돈이 새는 패턴", "유리한 흐름" 톤까지만

### 12-4. 공포 조장
- ❌ "이 날짜에 큰일 납니다"
- ❌ "조심하지 않으면 큰 손해를 봅니다"
- ✅ "이 시기에는 한 박자 천천히 가는 게 좋아요"
- ✅ "조심하면 좋은 흐름이 있어요"

## 13. 신규 상품 추가 시 카피 체크리스트

운명PT 에 새 상품을 추가할 때, 4개 위치를 **반드시** 동기화합니다.

- [ ] `supabase/migrations/000X_*.sql` — DB 시드 SQL
- [ ] `src/config/products.seed.ts` — 클라이언트 fallback 시드
- [ ] `src/config/product-copy.ts` — PRODUCT_COPY 매핑 (외부 노출 카피)
- [ ] `src/lib/saju/prompt.ts` — STYLE_BY_SLUG 매핑 (LLM 프롬프트 톤)

추가로 외부 콘텐츠가 그 상품을 참조한다면:
- [ ] `src/app/contents/page.tsx` — productSlug, productLabel 추가
- [ ] `src/app/contents/[category]/page.tsx` — productSlug, productName, productCta 추가

### 13-1. 톤 일관성 검수
- [ ] 외부 라벨에 "사주/점/운명/운세" 없음
- [ ] 상품 페이지 헤드라인이 외부 톤과 자연스럽게 연결됨
